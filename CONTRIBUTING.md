# How to contribute
If you would like to contribute to this project, take a look at the issues posted.  Issue fixes are resolved via pull request  Easier tasks for new contributors will be labeled with a good first issue tag.  If you would like to request a feature, read the guidlines and create an issue if your feature request qualifies.
# Code style
Follow the code style guide.  Include doc comments for your functions based on the templates found in doc.rs.
# Testing requirements
Test coverage requirements are not strict.  Just make sure you do not commit breaking changes.
# Documentation standards
Include doc comments for functions larger then the maximum size found in doc.rs.
# Bug reports 
Report bugs via our third party forum at virust/thirdpartyforum.com.
# Community Guidlines
Keep discussions in issues respectful.
